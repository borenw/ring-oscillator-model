1 · The idea in one line
Each inverter = 180° DC inversion + a pole lag φ(f) = −atan(f/f_p) that only reaches −90° at f → ∞. Around a loop of N stages the pole lag must top up the DC inversions to a multiple of 360°. A 4-stage ring needs a full −360° from the poles → only at infinity (where gain = 0) → it can't oscillate. A 3-stage ring needs only −180° → reached at a finite frequency where gain is still > 1 → it oscillates at f_osc = √3·f_p.

5 · 🐍 Python model source
ring_osc_model.py 
"""
Ring-oscillator phase model  (Barkhausen / phase-shift view)
============================================================
Reproduces the two hand-sketch scenarios:

  (A) EVEN 4-stage ring  -> "356 deg, not enough" : never oscillates
  (B) ODD  3-stage ring  -> phase sum crosses -180 deg at f_osc : oscillates

Model of ONE inverting gain stage (single dominant pole):

        H(jw) = -A0 / (1 + j*w/wp)

    - the "-" sign  = 180 deg DC inversion (frequency independent)
    - the pole      = extra lag  phi_pole(w) = -atan(w/wp)
      which only reaches -90 deg as w -> infinity.

Loop (N identical stages):  L = H^N
Oscillation (Barkhausen):   angle(L) = -360 deg  AND  |L| >= 1

    N inversions give N*180 deg.  The pole lag must supply the rest.
      N=3 : 540 deg  ->  need 3*(-60 deg) = -180 deg  (reachable, finite f)
      N=4 : 720 deg  ->  need 4*(-90 deg) = -360 deg  (only at inf -> fails)
"""

import numpy as np

# ---- stage parameters (popular / realistic numbers) --------------------
A0   = 4.0            # DC gain per stage (V/V)  -> 20*log10(4) = 12.04 dB
fp   = 1.0e9          # dominant pole of one inverter stage = 1 GHz
DEG  = 180.0 / np.pi

# frequency sweep: 0.01*fp ... 100*fp  (log spaced)
f  = np.logspace(np.log10(0.01*fp), np.log10(100*fp), 1500)
w  = 2*np.pi*f
wp = 2*np.pi*fp
x  = w/wp                     # normalized frequency f/fp

# ---- one stage ---------------------------------------------------------
mag_stage_dB   = 20*np.log10(A0/np.sqrt(1 + x**2))       # |H| in dB
phi_pole_stage = -np.arctan(x)*DEG                       # pole lag only, deg (0 -> -90)

# =======================================================================
# (A) EVEN 4-STAGE RING
# =======================================================================
N4 = 4
phi_pole_4 = N4*phi_pole_stage          # cumulative pole lag of 4 stages (0 -> -360)
# required cumulative pole lag to oscillate for the even ring = -360 deg
# -> impossible at finite f. Show how close we get at top of sweep.
f_hi           = np.tan(np.deg2rad(89))*fp   # probe freq giving 89 deg/stage (matches sketch)
x_hi           = f_hi/fp
phi_stage_hi   = -np.arctan(x_hi)*DEG    # ~ -87.1 deg  ("89-ish, not 90")
phi_4_hi       = N4*phi_stage_hi         # ~ -356 deg   ("not enough")

# =======================================================================
# (B) ODD 3-STAGE RING  (decomposed as A12 = 2 stages, A3 = 1 stage)
# =======================================================================
N3 = 3
phi_A12 = 2*phi_pole_stage               # two-stage cascade pole lag  (0 -> -180)
phi_A3  = 1*phi_pole_stage               # one-stage pole lag          (0 -> -90)
phi_sum = phi_A12 + phi_A3               # total pole lag = 3 stages    (0 -> -270)

# Oscillation: total pole lag = -180 deg  ->  atan(x) = 60 deg  ->  x = tan60 = sqrt(3)
x_osc   = np.tan(np.deg2rad(60))         # = sqrt(3) = 1.7320
f_osc   = x_osc*fp                        # = sqrt(3) * fp  = 1.732 GHz
# per-stage phase & loop gain at f_osc
phi_stage_osc = -np.arctan(x_osc)*DEG                    # -60.0 deg exactly
mag_stage_osc = A0/np.sqrt(1 + x_osc**2)                 # |H| at f_osc  = A0/2
loop_gain_osc = mag_stage_osc**N3                        # |L| at f_osc
Amin          = 2.0                                      # min DC gain/stage for 3-ring

# magnitude curves for the decomposition plot
mag_A12_dB = 2*mag_stage_dB
mag_A3_dB  = 1*mag_stage_dB

if __name__ == "__main__":
    print("=== stage ===")
    print(f"A0 = {A0}  ({20*np.log10(A0):.2f} dB),  fp = {fp/1e9:.3f} GHz")
    print("\n=== (A) even 4-stage ring ===")
    print(f"per-stage lag at {f_hi/1e9:.0f} GHz = {phi_stage_hi:6.2f} deg  (-> -90 only at inf)")
    print(f"4-stage cumulative lag         = {phi_4_hi:6.2f} deg  (need -360 -> NOT ENOUGH)")
    print("\n=== (B) odd 3-stage ring ===")
    print(f"x_osc = f_osc/fp = tan(60) = {x_osc:.4f}")
    print(f"f_osc = {f_osc/1e9:.4f} GHz")
    print(f"per-stage phase at f_osc = {phi_stage_osc:.2f} deg")
    print(f"total pole lag at f_osc  = {phi_A12[np.argmin(abs(f-f_osc))]+phi_A3[np.argmin(abs(f-f_osc))]:.2f} deg (target -180)")
    print(f"|H| at f_osc = A0/2 = {mag_stage_osc:.3f}")
    print(f"|L| = |H|^3 at f_osc = {loop_gain_osc:.3f}  (>=1 oscillates)")
    print(f"min DC gain/stage for sustained osc = {Amin}")

Model: single dominant-pole inverter, H(jω) = −A₀/(1+jω/ω_p). Frequencies normalized to f_p = 1 GHz; charts use Plotly (log-frequency axis, cursor spike lines, unified hover).

Why 1.19 GHz (small-signal) ≠ 4.36 GHz (transient) — and the sky130 model is not wrong. Both numbers are correct; they answer different questions.
  • The loop-gain AC is a small-signal analysis, linearised at the switching threshold. The DC loop (closed through the 1 MH inductor) biases every inverter at Vtrip ≈ 0.87 V — verified in the .op. There both devices are saturated, so the output resistance is at its maximum (ro ≈ 30 kΩ measured), giving the lowest pole, fp ≈ 0.69 GHz. Three poles reach −180° at 1.73·fp = 1.19 GHz. That is the frequency at which oscillation starts from noise.
  • The real oscillation is large-signal / slewing. Once it swings rail-to-rail, each stage switches with its devices in triode (Ron ≈ 8 kΩ, ~4× lower than ro), so the delay is short (td = 38 ps) and fosc = 1/(2N·td) = 4.36 GHz. The ro/Ron ≈ 4 ratio is exactly the 3.7× frequency ratio.
  • So it is a physics distinction, not a model error. The sky130 nfet_01v8/pfet_01v8 devices are used correctly; a ring oscillator is inherently a large-signal circuit, and no single-bias small-signal AC can predict its steady-state frequency. The delay-based fosc = 1/(2N·td) is the right predictor — and it matches the transient to the digit.

Open-loop response of the helper cell — A1·A2 (series) ∥ A3. This is the building block the 4-helper oscillator (§7) is made of: two inverters in series, in parallel with a single helper A3 (outputs shorted, the "phase-averaging" node). Its open-loop amplitude & phase directly predict the oscillation frequency:

∠H reaches −180° at 2.97 GHz (with |H| = 13 dB > 0 dB) — very close to the 4-helper oscillator's transient fosc = 2.80 GHz (green line). The A12 path (2 poles) rolls off faster than A3 (1 pole), so the summed output swings from non-inverting at DC to fully inverting (−180°) at a finite frequency — that crossing is what sets where the ring built from these cells oscillates.

7 · 🔁 8-inverter "4-helper" ring — a 4-net, 4-phase oscillator

The hand sketch's "Result: 4 helpers A₃" circuit is not a simple chain — it has only four nets (α, β, γ, δ) wired symmetrically: a distance-1 ring α→β→γ→δ→α (A1, A2, A4, A5) plus four helper diagonals — α→γ (A3), γ→α (A3″), β→δ (A3′), δ→β (A3‴). Every net is driven by two inverter outputs (shorted together) and drives two — the "phase-averaging" the sketch is about. I built this exact 4-net topology in sky130 ngspice.

Correct topology: 4 nets, 8 inverters. Ring edges A1·A2·A4·A5 (blue/navy); helper diagonals A3·A3″ cross-couple α↔γ, A3′·A3‴ cross-couple β↔δ. Colors match the sketch.
Result: it oscillates — fosc = 2.80 GHz, rail-to-rail on all four nets, ~90° apart (a 4-phase oscillator). The mixed helper+ring paths give odd inversion counts (e.g. α→γ via A3 = 1, then γ→α via A4·A5 = 2, total 3 = inverting), so the network meets the oscillation condition at a finite frequency — slower than the 3-inverter ring (more stages / more delay).

Four rail-to-rail waveforms v(α), v(β), v(γ), v(δ), each ~90° apart. Measured fosc = 2.80 GHz (T = 357 ps).

Takeaway: the extra stages and helper diagonals lower fosc from 4.36 GHz to 2.80 GHz (~1.6×) and turn the 3-phase ring into a symmetric 4-phase oscillator — more inverters and more feedback paths buy lower frequency and more output phases, at the cost of area and power.

📄 ac_loopgain_3stage.spice — odd ring, open-loop AC
* ================================================================
*  sky130 3-stage ring oscillator  --  OPEN-LOOP loop-gain (AC)
*  Replicates the phase-shift Bode diagram with real transistors.
*
*  Loop-break method (inductor / coupling-cap):
*    - Lbrk closes the loop at DC, so every stage self-biases to its
*      own switching threshold (the true ring operating point), but
*      is OPEN at AC -> the loop is broken for the small-signal test.
*    - Cinj (huge) injects the AC stimulus into n1 at AC only.
*    - With V(n1)=1<0 at AC, the measured loop gain is T = V(n3o).
*  Oscillation (Barkhausen): |T| >= 1 where /_T = 0 deg (mod 360).
* ================================================================
.lib "__LIBPATH__" tt
.include "inv.spice"

.param SUP=1.8
Vdd vdd 0 {SUP}

* --- three inverter stages: n1 -> n2 -> n3 -> n3o ---
Xi1 n1  n2  vdd 0 inv
Xi2 n2  n3  vdd 0 inv
Xi3 n3  n3o vdd 0 inv

* --- loop break: DC-closed (bias), AC-open ---
Lbrk n3o n1  1e6
* --- AC injection into n1 (DC-blocked) ---
Cinj nac n1  1e6
Vac  nac 0   dc 0 ac 1

.control
  ac dec 60 1e6 5e11
  let tmag = db(v(n3o))
  let tph  = 180/PI * ph(v(n3o))
  wrdata ac_loopgain_3stage.txt tmag tph
  print "--- 3-stage open-loop scan done ---"
.endc
.end

📄 plot_results.py — figures from the ngspice data
#!/usr/bin/env python3
"""
Plot the sky130 ring-oscillator ngspice results.
Reads the wrdata .txt files produced by the .spice decks and writes PNGs.
Run after: ngspice -b ac_loopgain_3stage.spice  (etc.)
"""
import numpy as np
import matplotlib
matplotlib.use("Agg")
import matplotlib.pyplot as plt

BLUE, RED, GREEN, GREY = "#0052CC", "#DE350B", "#00875A", "#5E6C84"
plt.rcParams.update({"font.size": 11, "font.family": "DejaVu Sans",
                     "axes.grid": True, "grid.alpha": .35, "figure.dpi": 130})

def load(fn):
    d = np.loadtxt(fn)
    return d[:, 0], d[:, 1], d[:, 3]      # freq/time, colA, colB


# ---------- 1. AC loop gain, 3-stage (odd -> oscillates) ----------
f, mag, ph = load("ac_loopgain_3stage.txt")
ph = np.unwrap(np.radians(ph)) * 180/np.pi
# small-signal Barkhausen: phase = 0 deg (mod 360) with |T|>=1
i = np.argmin(np.abs(ph))

def crossings(t, v, th=0.9):
    out = []
    for k in range(1, len(v)):
        if v[k-1] < th <= v[k]:
            out.append(t[k-1] + (th-v[k-1])*(t[k]-t[k-1])/(v[k]-v[k-1]))
